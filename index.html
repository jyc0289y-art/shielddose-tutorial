§1. STL 파일 사전 시각화

STL = STereoLithography 3D 모델 파일. 차폐체/검출기 형상을 GEANT4에 입력하는 표준 방식.

1.1 STL 파일이란?

STL 파일은 3D 모델을 삼각형 면(facet) 모음으로 표현하는 형식:

  • 각 facet = 3 정점(vertex) + 1 outward normal vector
  • ASCII 형식: 사람이 읽을 수 있는 텍스트
  • Binary 형식: 컴팩트한 이진 (80 byte header + 4 byte facet count + 50 byte/facet)
  • 단위: STL 파일 자체는 단위 정보 없음 (정점 좌표가 mm/cm/inch 어느 단위인지 사용자가 알아야 함)
  • 본 튜토리얼 STL은 모두 mm 단위 (Onshape/Fusion360 등 CAD 기본)

§3. cc 파일 export — GUI에서 전통 GEANT4 application 생성

우리 GUI의 핵심 혁신 #3. 사용자가 GUI에서 입력한 시뮬 조건을 표준 GEANT4 application 소스 패키지로 자동 변환.

3.1 cc export란?

일반 GEANT4 사용자는 시뮬을 위해 cc 파일을 직접 작성해야 합니다:

  • DetectorConstruction.cc — 차폐/검출기 형상 정의
  • PrimaryGeneratorAction.cc — 핵종/광원 정의
  • main.cc — application 진입점
  • CMakeLists.txt — 빌드 설정
  • run.mac — 실행 매크로

각 파일 작성에 수 시간~수 일 소요. 우리 GUI는 클릭 한 번으로 이 모든 파일을 자동 생성합니다.

4.1 main.cc — application 진입점

🎯 이 파일이 하는 일

GEANT4 시뮬레이션을 시작하는 메인 프로그램. 두뇌(RunManager) + 검출기 + 물리 모델 + 광원 액션 모두 등록하고 실행.

/// \file main.cc
/// \brief Auto-generated GEANT4 application entry (from ShieldDose GUI v3)

// ─── 1) GEANT4 핵심 헤더 ───
#include "G4RunManagerFactory.hh"   // 시뮬 전체를 관리하는 두뇌 (RunManager) 생성기
#include "G4UImanager.hh"           // 사용자 명령 (/run/beamOn 등) 입력 매니저
#include "G4VisExecutive.hh"        // 시각화 매니저 (3D 화면 표시)
#include "G4UIExecutive.hh"         // 인터랙티브 UI (Qt 콘솔)
#include "FTFP_BERT.hh"             // 표준 물리 모델 패키지 (전자기 + 강한 상호작용)

// ─── 2) 우리가 작성한 클래스 헤더 ───
#include "DetectorConstruction.hh"  // 차폐 + 검출기 형상 정의 (§4.3)
#include "ActionInitialization.hh"  // 광원 + 이벤트 액션 등록 (§4.7)

int main(int argc, char** argv) {
    // ─── 3) 시뮬 두뇌 생성 ───
    // G4RunManagerFactory: GEANT4의 핵심 객체. 시뮬 전체를 통제
    auto* runManager = G4RunManagerFactory::CreateRunManager(G4RunManagerType::Default);

    // ─── 4) 검출기/물리/액션 등록 (순서 중요) ───
    runManager->SetUserInitialization(new DetectorConstruction());  // 차폐+검출기 형상
    runManager->SetUserInitialization(new FTFP_BERT);                // 물리 모델 (감마/전자/중성자 등)
    runManager->SetUserInitialization(new ActionInitialization());   // 광원 및 액션

    // ─── 5) 초기화 — 위 등록된 것들로 시뮬 환경 구축 ───
    runManager->Initialize();

    // ─── 6) 시각화 매니저 ───
    G4VisManager* visManager = new G4VisExecutive;
    visManager->Initialize();
    G4UImanager* UImanager = G4UImanager::GetUIpointer();

    // ─── 7) 모드 분기 ───
    if (argc == 1) {
        // 인자 없음 → 인터랙티브 모드 (사용자가 콘솔에 명령 입력)
        G4UIExecutive* ui = new G4UIExecutive(argc, argv);
        UImanager->ApplyCommand("/control/execute macros/run.mac");  // 기본 매크로 실행
        ui->SessionStart();   // Qt UI 시작 — 사용자가 닫을 때까지 대기
        delete ui;
    } else {
        // 인자 있음 → 배치 모드 (매크로 실행 후 자동 종료)
        G4String command = "/control/execute ";
        G4String fileName = argv[1];  // 첫 인자 = 매크로 파일 경로
        UImanager->ApplyCommand(command + fileName);
    }

    // ─── 8) 정리 ───
    delete visManager;
    delete runManager;
    return 0;
}

4.3 src/DetectorConstruction.cc — 차폐/검출기 형상 구현

🎯 이 파일이 하는 일

실제 World + Pb 차폐 + water 검출기를 G4Box(직육면체) 형상으로 만들고 World에 배치. 이 파일이 GUI 입력값(Pb 5mm @ z=10cm, water 5×5×1cm @ z=100cm)을 코드로 표현한 것.

/// \file DetectorConstruction.cc
/// \brief Auto-generated from ShieldDose GUI v3 SimConfig

#include "DetectorConstruction.hh"
#include "G4NistManager.hh"              // NIST 표준 재료 라이브러리 (G4_Pb, G4_WATER 등)
#include "G4Box.hh"                       // 직육면체 형상
#include "G4PVPlacement.hh"               // 형상을 World에 배치
#include "G4SystemOfUnits.hh"             // 단위 (cm, mm, MeV 등)
#include "G4MultiFunctionalDetector.hh"   // 다중 측정 검출기 (dose, fluence 등)
#include "G4PSDoseDeposit.hh"             // 선량 침적 측정 (J/kg = Gy)
#include "G4SDManager.hh"                 // Sensitive Detector 관리자

DetectorConstruction::DetectorConstruction() {}

G4VPhysicalVolume* DetectorConstruction::Construct() {
    auto* nist = G4NistManager::Instance();   // NIST 재료 매니저 인스턴스

    // ═══════════════════════════════════════════
    // 1) World — 시뮬 공간 전체 (±5m air 박스)
    // ═══════════════════════════════════════════
    auto* worldMat = nist->FindOrBuildMaterial("G4_AIR");
    // G4Box("이름", 반-x, 반-y, 반-z)  ← 반-크기 사용 (관례)
    auto* worldSolid = new G4Box("World", 5.0*m, 5.0*m, 5.0*m);
    auto* worldLogical = new G4LogicalVolume(worldSolid, worldMat, "World");
    auto* worldPhys = new G4PVPlacement(nullptr,                // 회전 없음
                                         G4ThreeVector(),       // 원점 (0,0,0)
                                         worldLogical,
                                         "World",
                                         nullptr,                // mother = 없음 (최상위)
                                         false, 0, true);

    // ═══════════════════════════════════════════
    // 2) Shield — 5mm Pb 슬랩 차폐 (GUI 입력값 자동 주입)
    // ═══════════════════════════════════════════
    auto* shieldMat = nist->FindOrBuildMaterial("G4_Pb");        // NIST 표준 납
    // G4Box: 50×50×0.25 cm = 100×100×5 mm (full size)
    // GUI 입력 Thickness 5mm → 반-z = 0.25 cm
    auto* shieldSolid = new G4Box("Shield", 50*cm, 50*cm, 0.25*cm);
    auto* shieldLogical = new G4LogicalVolume(shieldSolid, shieldMat, "Shield");
    // Position z = 10 cm — 선원에서 10cm 떨어진 곳에 차폐 배치
    new G4PVPlacement(nullptr,
                       G4ThreeVector(0, 0, 10*cm),
                       shieldLogical, "Shield", worldLogical,
                       false, 0, true);

    // ═══════════════════════════════════════════
    // 3) Detector — 5×5×1 cm water 검출기 (GUI 입력값)
    // ═══════════════════════════════════════════
    auto* detMat = nist->FindOrBuildMaterial("G4_WATER");
    // G4Box: 2.5×2.5×0.5 cm = 5×5×1 cm (full size)
    auto* detSolid = new G4Box("Detector", 2.5*cm, 2.5*cm, 0.5*cm);
    fDetectorLogical = new G4LogicalVolume(detSolid, detMat, "Detector");
    // Position z = 100 cm = 1m — 선원에서 1m 거리
    new G4PVPlacement(nullptr,
                       G4ThreeVector(0*cm, 0*cm, 100*cm),
                       fDetectorLogical, "Detector", worldLogical,
                       false, 0, true);

    return worldPhys;
}

// ═══════════════════════════════════════════
// Sensitive Detector — 검출기에서 dose 측정 활성화
// ═══════════════════════════════════════════
void DetectorConstruction::ConstructSDandField() {
    // MultiFunctionalDetector: 여러 측정 항목 등록 가능
    auto* mfd = new G4MultiFunctionalDetector("DetectorSD");
    G4SDManager::GetSDMpointer()->AddNewDetector(mfd);

    // G4PSDoseDeposit: dose = energy_deposit / mass [Gy = J/kg]
    mfd->RegisterPrimitive(new G4PSDoseDeposit("Dose"));

    // Detector 논리 볼륨에 SD 할당 → 모든 이벤트마다 dose 자동 계산
    if (fDetectorLogical) SetSensitiveDetector("Detector", mfd);
}

  set(_g4_args ${ARGN})

    list(GET _g4_args 1 _g4_doc)
  elseif(_g4_argc EQUAL 1)

    set(_g4_doc "")
  else()
    set(_g4_type FILEPATH)
    set(_g4_doc "")
  endif()
  if(DEFINED ${_name})
    if(NOT ("${${_name}}" STREQUAL "${_value}"))
      message(WARNING "Value of '${_name}' does not match Geant4 build-time value")
    endif()
  else()
    set(${_name} "${_value}" CACHE ${_g4_type} "${_g4_doc}")
  endif()
endmacro()

§9. 물리 개념 미니 박스

9.1 H*(10) — Ambient Dose Equivalent (ICRP-74)

방사선 작업 환경의 실효 선량 추정. ICRP Publication 74 표준.

  • 정의: 30cm × 30cm × 30cm ICRU 구면 팬텀 10mm 깊이 dose
  • 실용 단위: μSv/h
  • 변환: air kerma × h_K 계수 (Cs-137: 1.20, Co-60: 1.15)

9.2 water dose vs air kerma (PIT-003)

두 양은 다름. 단위 환산 필수.

  • water dose (D_water): water 검출기 안의 에너지 침적 / 질량 (Gy = J/kg)
  • air kerma (K_air): 동일 위치에 air가 있다면 받을 kerma
  • 변환: K_air = D_water × (μ_en/ρ)_air / (μ_en/ρ)_water
  • Cs-137 0.662 MeV: ratio ≈ 1.108 (NIST XCOM)
  • ShieldDose는 자동 변환 (PIT-003 fix 적용)

9.3 CPE — Charged Particle Equilibrium (PIT-001)

검출기 안에서 광전/Compton으로 생성된 전자의 range가 검출기 두께보다 작아야 dose 정확.

  • Cs-137 0.662 MeV electron range ≈ 2 mm in water
  • 최소 두께: 3× range = 6 mm 권장. 본 검출기 1 cm = 5× → ✓ PASS
  • 두께 부족 시 dose 과소예측 (~46% 부족 가능, P2 실측 사례)

9.4 SA ratio — Solid Angle / Distance² (PIT-006)

점선원 가정의 유효성. SA ratio = √(detector_area) / distance < 0.1 권장.

  • 본 시뮬: √(5×5) / 100 = 5/100 = 0.05 ✓ PASS
  • 위반 시 (예: 30cm + 20×20 검출기 → 0.33): 점선원 역제곱 환산 부정확 → 시뮬 차단

9.5 Buildup factor (PIT-008)

broad-beam 차폐에서 산란 광자 보정. transmission = exp(-μρt) × B(μρt, E).

  • ANS-6.4.3-1991 빌드업 데이터 ShieldDose 내장
  • Pb 5mm Cs-137: B ≈ 1.2~1.5 (단순 exp 대비 보정)
  • MC 시뮬은 자동 처리 (산란 트래킹). 해석적 계산만 별도 보정 필요

9.6 Gamma Constant Γ

1 Ci 점선원 1m 거리 air kerma rate (또는 exposure rate).

  • 단위: R·m²/(Ci·h) (전통) 또는 mSv·m²/(GBq·h) (SI)
  • Cs-137: Γ = 0.330 R·m²/(Ci·h) (ICRP Pub 21)
  • Co-60: Γ = 1.300 R·m²/(Ci·h)
  • 점선원 무차폐 1m H*(10): D_rate ≈ Γ × A / r² × h_K

9.7 ISS-087 단위 mis-conversion 학습 사례

P2.17.1.0에서 발견된 결정적 버그:

  • 증상: STL 검출기 dose가 슬랩 검출기의 151× 큼
  • 가설 매트릭스 검증:
    • (a) G4TessellatedSolid voxel 부피 부정확 — STLParser ratio=1 측정으로 부정
    • (g) 슬랩+STL 이중 SD scoring — 검증 시뮬로 부분 원인 6%
    • (n) 매크로 단위 mis-conversion — 진짜 주범 확정 ★
  • 원인: MacroGenerator.cc:144-152가 shieldSTL_X_cm (cm 값) + shieldSTL_Unit ("mm"/"cm" 문자열) 두 의미 다른 단위를 한 매크로 라인에 mixed 출력 → 사용자 "mm" 선택 시 위치 1/10 mis-conversion (10cm → 1cm) → 역제곱 100× dose 증폭
  • fix (v0.9): MacroGenerator 위치 라벨을 항상 "cm" 고정 (SimConfig 필드가 cm 단위)
  • 교훈: 단위는 매크로 + 코드 + GUI 세 곳 모두 일관 유지 필수

10.2 매크로 단위 (ISS-087 fix 이전 vs 이후)

ISS-087 fix 이후 (v0.9+): 매크로 위치 라벨 항상 "cm" 고정. STL 메시 단위 (mm/cm/inch) 콤보는 메시 scale 처리만 (위치와 무관)

본 튜토리얼 단계 따라하면 모두 fix 적용된 v0.9 이후 동작.

10.3 빌드 실패 — geant4_set_and_check_package_variable 매크로 에러

증상: CMake Error: geant4_set_and_check_package_variable Macro invoked with incorrect arguments

원인: GEANT4 11.1.2 + CMake 4.x 호환성 (GEANT4 PackageCache.cmake가 3 인자만 전달)

해결: CMakeLists.txt에 macro override 추가 (§4.8 본문 참고). v0.9+의 cc export는 자동 포함.

10.4 STL 단위 자동 추정 한계 (PIT-026)

증상: 50mm 박스 STL을 GUI에 임포트 → 자동 추정 "cm" 표시 (실제는 mm)

원인: 휴리스틱 (bbox < 100 → cm 추정, > 100 → mm 확정). 작은 mm 모델은 cm으로 추정될 수 있음

해결: 단위 콤보 수동 "mm"로 변경. ISS-087 fix 후 매크로 위치는 항상 cm이라 mismatch 없음

10.5 GEANT4 vis manager에서 3D 안 보임

/vis/drawVolume      # geometry 그리기
/vis/viewer/flush    # 화면 갱신

10.6 Cf-252 mixed-field 시뮬

Cf-252는 자발 핵분열로 중성자 + 감마 동시 방출. Neutron HP physics 필요.

ShieldDose는 매크로의 /physics/ 명령 pre-scan으로 자동 활성:

/physics/biasing both           # γ + neutron 둘 다 importance biasing
